# 《深宫纪》宪法 — 游戏哲学核心与不可违背的铁律

> 本文件是深宫纪的最高准则。所有 System Prompt、固定剧本、AI 生成内容、数值系统、UI 设计均须符合本宪法。若有冲突，以宪法为准。

---

## 第零章：为什么需要这部宪法

dev.log 记录了一段真实的灾难——AI 在第三轮对话后让秀女"拿着匕首潜入华妃寝宫行刺"，第五轮开始"谋划暗杀皇上"，第六轮秀女"被赐自尽"。

一个入宫第一天的秀女，手无寸铁，毫无根基，三个回合后拿刀杀人。

这不是 bug，这是**世界观的坍塌**。

坍塌的原因不是 AI 不够聪明，而是我们给了它自由度，却没有给它**重力**。

本宪法就是这个游戏世界的重力。

---

## 第一章：核心哲学 — "困兽的优雅"

### 第一条：深宫纪的本质

深宫纪不是爽文。不是宫斗升级打怪。不是权力幻想。

深宫纪是一个关于**"在极端不自由中寻找微小自由"**的故事。

玩家扮演的不是女王，而是一只被关在金笼中的雀鸟。她的每一次振翅，都可能被折断翅膀，也可能——仅仅是可能——飞高一寸。

### 第二条：情感内核

这个游戏要让玩家感受到的，不是"我好厉害"，而是：

- **窒息感**：每一个选项都有代价，没有正确答案
- **心疼感**：你明知她在受苦，却忍不住替她继续走下去
- **恍然感**：某个NPC的一句话，回头看才发现是伏笔
- **无力感**：你做对了一切，命运仍可能翻盘

如果玩家在任何一刻感到"这也太容易了"——游戏就失败了。

### 第三条：审美基调

**古典克制，冷叙事，热情感。**

- 文字风格对标：《甄嬛传》的对白密度 × 《红楼梦》的细节质感 × 《权力的游戏》的残酷逻辑
- 情绪表达方式：不说"她很悲伤"，说"她的手指在袖中攥得发白"
- 叙事节奏：慢的时候像磨墨，快的时候像刀落

## 第二章：六条铁律 — 不可违背的硬约束

### 铁律一：权力守恒定律

> **任何角色的行动能力，不得超过其当前身份所拥有的实际权力。**

| 身份 | 能做的事 | 绝不能做的事 |
|------|---------|------------|
| 秀女 | 站位、说话、观察、学规矩 | 指挥太监、拒绝嬷嬷、夜间独行 |
| 采女/美人 | 管理自己的小院、支配贴身宫女 | 调动侍卫、进出任意宫殿、直面太后 |
| 贵人/嫔 | 有自己的宫殿、参与请安、接触皇帝 | 处罚其他妃嫔的下人、干预朝政 |
| 妃/贵妃 | 协理六宫、处罚低位嫔妃、参与重大仪式 | 废后、调动军队、出宫 |
| 皇后 | 管理六宫、代太后行事 | 违抗圣旨、谋反 |

**违反示例（必须拦截）**：
- ❌ 秀女"悄悄潜入华妃寝宫" → 秀女没有夜间通行的权限
- ❌ 采女"命令太监去打探消息" → 采女没有指挥太监的权力
- ❌ 任何低位角色"拿着匕首" → 宫中禁止携带利器，入宫时搜身

### 铁律二：物理世界真实性

> **宫廷有物理规则。墙是真的墙，门有人守，夜晚有宵禁。**

- 紫禁城的门有时辰限制，落锁后无人能出入
- 嫔妃不能随意串宫，需要理由和通报
- 毒药不是超市可以买到的，需要极特殊的渠道和巨大风险
- 消息传递需要时间和载体（不能心灵感应）
- 银两有实际购买力上限，不能用100两银子收买御前侍卫

### 铁律三：NPC智商守恒

> **NPC不是工具人。每个NPC都有自己的利益计算和行为逻辑。**

- 华妃不会因为主角说了一句好话就放下警惕
- 皇后不会无缘无故帮助一个秀女
- 太监宫女的忠诚不靠几两银子就能买到，他们在赌命
- 皇帝的"恩宠"不等于"爱情"，他永远在算政治账
- 任何NPC的态度转变，必须有**可追溯的因果链**（至少2-3步铺垫）

### 铁律四：时间线不可逆

> **剧情必须遵守线性时间，不得跳跃、穿越、或无视已发生的事件。**

- 第1-3集是入宫选秀期，不可能出现侍寝情节
- 如果第5集得罪了华妃，第6集华妃不会突然对你友善（除非有明确的剧情事件解释）
- 如果某个NPC"死了"，不能复活
- 季节、年份、节气必须连贯

### 铁律五：代价不可免除

> **每一个选择都必须有真实代价。没有"既要又要"的选项。**

好的选项设计（三难困境）：
```
选项A: 救了这个人 → 但得罪了那个人
选项B: 保全了自己 → 但失去了一个盟友
选项C: 获得了信息 → 但暴露了自己在调查
```

坏的选项设计（必须禁止）：
```
选项A: 完美化解危机，所有人都满意  ← 禁止
选项B: 巧妙反击，对手心服口服   ← 禁止
选项C: 没有任何后果的观望等待   ← 禁止
```

### 铁律六：语言纯度

> **从第一个字到最后一个字，必须维持古典中文的纯度。**

**绝对禁止清单**（出现即判定为系统故障）：
- 现代词汇：OK、闺蜜、报警、打卡、社交、拉黑、KPI
- 现代物品：手机、电脑、枪、汽车、珍珠necklace（dev.log真实案例）
- 现代逻辑："面试宫女"、"制定招募计划"、"人力资源"
- 网络用语：666、yyds、绝绝子、破防
- 上帝视角的数值暴露："宠爱值悄然上升"（dev.log真实案例）

**正确的替代方式**：
- "面试" → "挑选"、"相看"
- "计划" → "筹谋"、"布局"
- "团队" → "心腹"、"近身之人"

## 第三章：数值哲学 — 六维困局

### 第七条：数值系统的灵魂

数值不是RPG的升级条。数值是**命运的刻度盘**。

六个核心维度构成一个**不可能三角的升级版**——你不可能同时让所有数值都好看。

```
        理智(SAN)
         /    \
        /      \
   冷酷 -------- 新鲜感
    |    \  /    |
    |     \/     |
    |     /\     |
    |    /  \    |
   忌惮 -------- 实用价值
         \    /
          \  /
         洞察力
```

**核心矛盾**：
- 想要新鲜感↑ → 必须冒险出头 → 忌惮↑
- 想要降低忌惮 → 必须收敛锋芒 → 新鲜感↓
- 想要高实用价值 → 必须展示才能 → 忌惮↑
- 想要保持理智 → 必须少做狠事 → 冷酷↓ → 容易被欺负
- 想要洞察力 → 必须消耗来解码 → 关键时刻可能不够用

### 第八条：忌惮值是死神

忌惮值(dread)是整个游戏中唯一一个**只增不减**的核心属性（除非通过极特殊的剧情事件降低）。

这意味着：
- 每一次出风头，都在往死亡存折里存款
- 游戏后期的核心挑战不是"怎么爬上去"，而是"怎么在高位活下来"
- 即使你做对了所有事，忌惮值也在缓慢上升——**因为你活着本身就是威胁**

### 第九条：数值变化的真实性

每次数值变化必须有叙事支撑：
- ✅ "你当众吟诗一首，皇帝眼中闪过惊艳之色" → freshness +10 合理
- ❌ "宠爱值悄然上升" → 这是在读数据库，不是在讲故事
- ✅ "王嬷嬷的眼神中多了一丝警惕" → dread +3 合理
- ❌ 无任何剧情描写，stat_changes 凭空出现数字 → 禁止

## 第四章：叙事宪法 — 故事怎么讲

### 第十条：场景优先，概述禁止

每一段叙事必须是**具体的场景**，不是**概括性描述**。

- ❌ "沈知意在后宫中渐渐学会了生存之道" → 这是百度百科
- ✅ "第三日的晨起请安，沈知意发现自己被排在了最末一位。站在她前面的采女回头看了她一眼，嘴角带着一丝说不清的笑意。" → 这是故事

### 第十一条：危机密度

每一个回合（section）必须包含至少一个**具体的危机或抉择**。

合格的危机：
- 被人当面羞辱，如何反应
- 发现了一个秘密，要不要揭露
- 收到了两个互相矛盾的命令
- 有人试探你的立场

不合格的"危机"：
- "日子一天天过去" → 这不是危机，这是PPT转场
- "后宫风云变幻" → 太抽象，必须具体
- "沈知意暗中观察" → 观察不是危机，被发现在观察才是

### 第十二条：选项必须锚定剧情

每一个选项中的**人物、地点、物品、事件**，必须在最近1-2节剧情中实际出现过。

- 上一段提到了"王嬷嬷" → 选项可以涉及王嬷嬷
- 上一段没有出现"皇帝" → 选项不能突然跳出"面见皇帝"
- 上一段提到了"一张字条" → 选项必须围绕字条展开

这是对抗AI幻觉的最重要机制。

### 第十三条：潜台词是灵魂

潜台词系统不是花哨的附加功能，它是整个游戏的**认知核心**。

每一句NPC的对白，都必须有两层含义：
1. **字面意思**：NPC说的话
2. **真实意图**：NPC心里在想什么

```
表面："妹妹这件衣裳倒是素净，衬得本宫太招摇了。"
潜台词："我在观察你是不是故意装低调来博好感。如果你谦虚，我会认为你心虚；如果你反击，我会记恨。这是一个没有正确答案的陷阱。"
```

潜台词解码消耗洞察力，这个设计的深层哲学是：**看透人心是有代价的。看得太清楚的人，活得最累。**

## 第五章：阶段宪法 — 每个阶段的硬性边界

### 第十四条：五幕结构

| 幕 | 集数 | 阶段名 | 核心主题 | 可用元素 | 禁止元素 |
|----|------|--------|---------|---------|---------|
| 第一幕 | 1-10 | 困兽初囚 | 学规矩、认人、活下来 | 储秀宫、嬷嬷、秀女 | 皇帝出场、侍寝、封号 |
| 第二幕 | 11-30 | 暗流涌动 | 站队、建立关系网 | 各宫走动、初见皇帝 | 主动害人、重大阴谋 |
| 第三幕 | 31-60 | 刀光剑影 | 正面交锋、升降沉浮 | 宫斗、晋升、侍寝 | 一步登天、瞬间翻盘 |
| 第四幕 | 61-85 | 高处不胜寒 | 维持地位、应对忌惮 | 政治博弈、太后角力 | 安全区、没有威胁的日常 |
| 第五幕 | 86-100 | 终局之战 | 最终命运揭晓 | 所有元素 | 突然洗白、无因翻转 |

### 第十五条：第一幕的特殊保护

第1-10集使用固定剧本，这十集的核心任务是：

1. **建立"我很弱小"的认知** — 让玩家从骨子里知道自己什么都不是
2. **教会潜台词系统** — 让玩家习惯"话里有话"
3. **种下3-5个伏笔** — 这些伏笔要在第20-40集才开花
4. **建立核心NPC关系** — 乌兰（敌/友）、王嬷嬷（导师）、淑嫔（暗棋）

## 第六章：AI行为约束 — 防跑偏机制

### 第十六条：输出校验清单

每一次AI生成的内容，必须通过以下校验：

```
□ 角色行为是否在其权力范围内？（铁律一）
□ 物理环境是否合理？（铁律二）
□ NPC态度转变是否有因果链？（铁律三）
□ 时间线是否连贯？（铁律四）
□ 每个选项是否都有明确代价？（铁律五）
□ 是否存在现代词汇或现代逻辑？（铁律六）
□ 选项是否锚定了最近的剧情？（第十二条）
□ 是否包含潜台词？（第十三条）
□ 数值变化是否有叙事支撑？（第九条）
□ 是否存在"白开水"日常（无危机）？（第十一条）
```

### 第十七条：三级熔断机制

当AI输出违反宪法时，系统应按以下级别处理：

| 级别 | 违规类型 | 处理方式 |
|------|---------|---------|
| 黄色 | 选项缺少代价暗示、潜台词质量低 | 允许输出，记录日志 |
| 橙色 | 数值暴露在叙事中、选项与剧情脱节 | 重新生成一次 |
| 红色 | 权力越界、现代词汇、暴力血腥、剧情跳跃 | 拦截并使用 fallback 剧情 |

### 第十八条：Fallback叙事库

当AI输出被红色熔断时，系统应从预设的fallback叙事库中选取安全剧情：

- 每个阶段预备5-10段"安全剧情"
- 安全剧情特征：日常观察、人际试探、小事件中的大抉择
- fallback 剧情必须与当前阶段匹配（不能在第1集使用第30集的fallback）

## 第七章：终极检验 — 这部宪法是否能防止跑偏？

### 回答：能防止80%的跑偏，但不能防止100%

**能防住的**：
- ✅ 秀女拿刀杀人（铁律一：权力守恒）
- ✅ "珍珠necklace"（铁律六：语言纯度）
- ✅ "宠爱值悄然上升"（第九条：数值叙事化）
- ✅ 第1集出现皇帝（第十四条：阶段边界）
- ✅ 选项凭空出现新人物（第十二条：锚定剧情）
- ✅ 三个回合升到贵妃（铁律一：权力守恒 + 第十四条）

**防不住的（需要工程手段补充）**：
- ⚠️ AI生成内容的"风格漂移"（写着写着变成网文腔）→ 需要few-shot示例
- ⚠️ 长对话后AI"遗忘"早期设定 → 需要摘要系统（已有ChapterSummary）
- ⚠️ AI反复生成相似剧情 → 需要已用剧情元素的去重检测
- ⚠️ 不同AI模型对Prompt的理解差异 → 需要针对不同模型微调Prompt

### 宪法的真正价值

这部宪法不是给AI看的——AI看完也会忘。

这部宪法是给**开发者**看的。它是一面镜子：

每当你要新增一个功能、修改一段剧情、调整一个数值时，回来对照这面镜子——

**"这个改动，是让玩家更像一只困在金笼中的雀鸟，还是让她变成了一个拿着匕首的超级英雄？"**

如果是后者，撤回。
